## 1. Comprehensive Illustration Catalog

### Category 1: Process & Flow Diagrams

#### 1.1 Linear Process Flow (Sequential)
- **Alternative Names**: Sequential process, step-by-step workflow, horizontal process
- **Industry Usage**: Essential (10/10 frequency) - Used across ALL industries
- **Primary Use Cases**:
  1. Customer journey mapping in marketing/sales
  2. Project phase visualization in consulting
  3. Manufacturing processes
  4. Software development workflows
  5. Onboarding processes
- **Visual Specifications**:
  - Shapes: Rectangles (rounded corners 8-12px), right arrows (20-30% of rectangle width)
  - Element count: 3-7 steps typical (max 10)
  - Layout: Horizontal left-to-right, equal spacing
  - Aspect ratio: 3:1 to 5:1 (wide format)
  - Color: Single color family with gradient OR distinct color per step
- **Content Structure**:
  - Title per step: 15-40 characters
  - Optional subtitle/description: 50-100 characters
  - Numbering: Sequential (1, 2, 3... or Step 1, Step 2...)
  - Icons: Optional, centered above or left of text
- **Styling Standards**:
  - Gradients: Yes - subtle within steps for depth (light to dark, 10-15% variation)
  - Shadows: Soft drop shadows (2-4px offset, 10-15% opacity)
  - Borders: Optional 1-2px stroke
  - Spacing: 20-30px between elements
- **Implementation**:
  - Difficulty: Simple
  - Approach: Template-based with parameterized steps
  - Estimated effort: 4-6 hours
  - Shape composition: 85% rectangles, 15% arrows
  - Custom SVG needed: No
- **Business Value Score**: 29/30
  - Frequency: 10/10 (used in virtually every presentation)
  - Applicability: 10/10 (all industries)
  - Importance: 9/10 (fundamental communication tool)

#### 1.2 Vertical Process Flow
- **Alternative Names**: Downward process, cascading workflow
- **Industry Usage**: Common (7/10 frequency) - Especially in project management, manufacturing
- **Primary Use Cases**:
  1. Decision trees
  2. Approval hierarchies
  3. Waterfall project phases
  4. Supply chain flows
- **Visual Specifications**:
  - Shapes: Rectangles, downward arrows
  - Element count: 3-8 steps typical
  - Layout: Vertical top-to-bottom
  - Aspect ratio: 1:2 to 1:3 (tall format)
- **Content Structure**:
  - Title per step: 15-40 characters
  - Description: 30-80 characters
  - Numbering: Optional
- **Styling Standards**:
  - Gradients: Yes
  - Shadows: Medium drop shadows (3-5px)
  - Borders: Rounded corners
- **Implementation**:
  - Difficulty: Simple
  - Approach: Template-based
  - Estimated effort: 4-6 hours
  - Shape composition: 85% rectangles, 15% arrows
  - Custom SVG needed: No
- **Business Value Score**: 24/30
  - Frequency: 8/10
  - Applicability: 8/10
  - Importance: 8/10

#### 1.3 Swimlane Diagram
- **Alternative Names**: Cross-functional flowchart, responsibility matrix flow
- **Industry Usage**: Common (8/10 frequency) - Consulting, operations, project management
- **Primary Use Cases**:
  1. Cross-departmental processes
  2. Responsibility mapping
  3. RACI workflows
  4. Complex project phases with multiple actors
- **Visual Specifications**:
  - Shapes: Horizontal lanes (rectangles), process boxes within lanes, arrows
  - Element count: 3-6 lanes, 4-12 process boxes
  - Layout: Horizontal lanes with left-side labels
  - Aspect ratio: 4:1 to 5:1 (very wide)
- **Content Structure**:
  - Lane labels: 10-30 characters (role/department names)
  - Process boxes: 15-50 characters
  - Arrows show sequence and handoffs
- **Styling Standards**:
  - Gradients: Minimal - lane backgrounds subtle
  - Shadows: Light shadows on process boxes
  - Borders: 1-2px lane separators, rounded process boxes
  - Colors: Different color per lane for clarity
- **Implementation**:
  - Difficulty: Moderate
  - Approach: Template-based with grid system
  - Estimated effort: 8-12 hours
  - Shape composition: 60% rectangles, 30% connecting lines, 10% arrows
  - Custom SVG needed: No
- **Business Value Score**: 25/30
  - Frequency: 8/10
  - Applicability: 8/10
  - Importance: 9/10

#### 1.4 Circular Process Flow (Cycle)
- **Alternative Names**: Circular workflow, continuous cycle, loop diagram
- **Industry Usage**: Common (8/10 frequency) - Quality management, continuous improvement
- **Primary Use Cases**:
  1. PDCA cycle (Plan-Do-Check-Act)
  2. Continuous improvement loops
  3. Iterative development cycles
  4. Feedback loops
  5. Product lifecycle
- **Visual Specifications**:
  - Shapes: Curved segments forming circle, directional arrows
  - Element count: 3-8 segments (4 most common)
  - Layout: Circular/radial from center
  - Diameter: 600-800px typical
- **Content Structure**:
  - Label per segment: 10-25 characters
  - Description: 30-60 characters
  - Arrows: Clockwise direction standard
- **Styling Standards**:
  - Gradients: Yes - within segments
  - Shadows: Medium for 3D funnel effect
  - Borders: 2-3px between segments
  - Colors: 4 distinct colors for 4-segment, gradient for continuous
  - Transparency: Optional for 3D effect
- **Implementation**:
  - Difficulty: Moderate
  - Approach: Template-based with arc calculations
  - Estimated effort: 10-14 hours
  - Shape composition: 70% arc segments, 20% arrows, 10% center hub
  - Custom SVG needed: Yes - for smooth arcs
- **Business Value Score**: 26/30
  - Frequency: 9/10
  - Applicability: 8/10
  - Importance: 9/10

#### 1.5 Timeline (Horizontal)
- **Alternative Names**: Project timeline, roadmap, chronological flow
- **Industry Usage**: Essential (9/10 frequency) - Universal across industries
- **Primary Use Cases**:
  1. Project milestones
  2. Company history
  3. Product roadmaps
  4. Strategic planning horizons
  5. Historical events
- **Visual Specifications**:
  - Shapes: Horizontal line with vertical markers, circles/diamonds for milestones
  - Element count: 5-12 milestones typical
  - Layout: Horizontal left-to-right with timeline base
  - Aspect ratio: 5:1 to 6:1 (very wide)
- **Content Structure**:
  - Date/period: 10-20 characters
  - Event title: 15-40 characters
  - Description: 30-80 characters (optional)
  - Alternating above/below line for readability
- **Styling Standards**:
  - Gradients: Minimal - solid colors preferred
  - Shadows: Light on milestone markers
  - Borders: 2-3px timeline spine
  - Colors: Timeline spine neutral, milestones colored
- **Implementation**:
  - Difficulty: Simple to Moderate
  - Approach: Template-based with configurable milestones
  - Estimated effort: 6-8 hours
  - Shape composition: 50% lines, 30% circles/diamonds, 20% text containers
  - Custom SVG needed: No
- **Business Value Score**: 28/30
  - Frequency: 9/10
  - Applicability: 10/10
  - Importance: 9/10

#### 1.6 Timeline (Vertical)
- **Alternative Names**: Vertical roadmap, chronological stack
- **Industry Usage**: Occasional (6/10 frequency)
- **Primary Use Cases**:
  1. Career progression paths
  2. Project phases in tall slides
  3. Historical narratives
- **Visual Specifications**:
  - Shapes: Vertical line with horizontal markers
  - Element count: 4-10 milestones
  - Layout: Vertical top-to-bottom
  - Aspect ratio: 1:3 to 1:4
- **Content Structure**:
  - Similar to horizontal timeline
  - Alternating left/right positioning
- **Styling Standards**:
  - Similar to horizontal timeline
  - Adjusted for vertical orientation
- **Implementation**:
  - Difficulty: Simple
  - Approach: Variant of horizontal timeline
  - Estimated effort: 3-4 hours (after horizontal exists)
  - Shape composition: 50% lines, 30% markers, 20% text
  - Custom SVG needed: No
- **Business Value Score**: 20/30
  - Frequency: 7/10
  - Applicability: 7/10
  - Importance: 6/10

#### 1.7 User Journey Map
- **Alternative Names**: Customer journey, experience map, touchpoint map
- **Industry Usage**: Essential (9/10) - Marketing, UX, product management
- **Primary Use Cases**:
  1. Customer experience analysis
  2. Digital product design
  3. Service blueprint
  4. Omnichannel mapping
- **Visual Specifications**:
  - Shapes: Horizontal timeline with swim lanes for stages, emotions, touchpoints
  - Element count: 5-8 journey stages, multiple touchpoints per stage
  - Layout: Multi-lane horizontal with stage columns
  - Aspect ratio: 4:1 to 5:1
  - Time axis: Quarters, months, or years
- **Content Structure**:
  - Stage names: 10-25 characters
  - Actions: 20-50 characters
  - Emotions: Icons or emoji + labels
  - Touchpoints: Icons with brief labels
  - Pain points: Red flags or indicators
- **Styling Standards**:
  - Gradients: Minimal
  - Shadows: Light
  - Borders: Clear lane separation
  - Colors: Emotion-coded (green=positive, red=negative, yellow=neutral)
- **Implementation**:
  - Difficulty: Moderate to Complex
  - Approach: Template-based with grid + icon system
  - Estimated effort: 12-16 hours
  - Shape composition: 40% rectangles/lanes, 30% icons, 30% connectors
  - Custom SVG needed: No
- **Business Value Score**: 26/30
  - Frequency: 8/10
  - Applicability: 8/10
  - Importance: 10/10 (critical for CX)

#### 1.8 Value Chain Diagram
- **Alternative Names**: Supply chain flow, value stream map
- **Industry Usage**: Common (7/10) - Manufacturing, operations, strategy
- **Primary Use Cases**:
  1. Operations analysis
  2. Cost structure mapping
  3. Supply chain visualization
  4. Business process optimization
- **Visual Specifications**:
  - Shapes: Large rectangles for activities, arrows for flow
  - Element count: 5-9 primary activities, 3-5 support activities
  - Layout: Horizontal primary chain, support activities above/below
  - Aspect ratio: 4:1
- **Content Structure**:
  - Activity name: 15-30 characters
  - Value contribution: Percentage or metric
  - Support functions labeled separately
- **Styling Standards**:
  - Gradients: Yes - to show value accumulation
  - Shadows: Medium depth
  - Borders: Clear separation between activities
  - Colors: Primary activities one color family, support activities another
- **Implementation**:
  - Difficulty: Moderate
  - Approach: Template-based
  - Estimated effort: 8-10 hours
  - Shape composition: 80% rectangles, 20% arrows
  - Custom SVG needed: No
- **Business Value Score**: 22/30
  - Frequency: 7/10
  - Applicability: 7/10
  - Importance: 8/10
